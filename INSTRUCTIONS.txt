=============================================================================
ISTRUZIONI PER ESEGUIRE IL PROGETTO
=============================================================================

0.5. üßπ (OPZIONALE) PULIZIA DATI
   Se hai nuovi file CSV nella cartella 'data/raw' e vuoi rigenerare i file puliti in 'data/processed'.
   
   1. Apri 'docker-compose.yml' e togli i commenti (##) alla sezione 'cleaning'.
   2. Esegui: docker-compose up cleaning
   3. (Opzionale) Ricommenta la sezione 'cleaning' nel docker-compose.yml.

-----------------------------------------------------------------------------

1. üü¢ AVVIARE IL DATABASE (Docker)
   Esegui questo comando per accendere il container del database in background.
   Il flag '-d' lo fa girare in background (detached).
   'db' specifica di avviare solo il servizio database.
   (stesso comando sia per powershell che wsl
   dato che "&" tecnicamente funziona, ma docker continua a stampare i log a schermo 
   e sembra che il terminale sia bloccato quando non lo √®
   usiamo allora il flag specifico di docker)

   COMMAND (Windows/WSL/Linux): docker-compose up -d db

-----------------------------------------------------------------------------

2. üêç ESEGUIRE LO SCRIPT PYTHON
   Questo comando avvia lo script che legge i CSV dalla cartella 'data/processed'
   e li carica nel database MySQL. Usa l'interprete Python nell'ambiente virtuale.
   
   COMMAND (Windows): .venv\Scripts\python src/main.py
   
   COMMAND (WSL/Linux): ./.venv/bin/python src/main.py
   (NOTA: Su WSL non puoi usare il .venv creato su Windows perch√© i file .exe non funzionano. 
    Devi ricreare l'ambiente: 'python3 -m venv .venv_wsl' -> 'source .venv_wsl/bin/activate' -> 'pip install -r requirements.txt')

-----------------------------------------------------------------------------

3. üîç CONTROLLARE LO STATO
   Verifica che il container del database sia attivo e funzionante.
   Dovresti vedere 'db_universita' nella lista.
   
   COMMAND (Windows): docker ps
   
   COMMAND (WSL/Linux): ps -e | grep docker

-----------------------------------------------------------------------------

4. üî¥ STOPPARE TUTTO
   Quando hai finito, spegni il database per liberare memoria e risorse.
   Questo ferma e rimuove i container (i dati rimangono salvi nel volume).
   
   COMMAND (Windows/WSL/Linux): docker-compose down

-----------------------------------------------------------------------------

5. üì∏ SALVARE LO STATO (SNAPSHOT)
   Se vuoi salvare i dati attuali nel file 'init.sql' per distribuire il progetto
   o per avere un backup:
   
   COMMAND (Windows): .\save_db_snapshot.bat
   
   oppure Doppio click su 'save_db_snapshot.bat'
